Add `map()`, a field instruction for shallow reactive collections - the field resolves at activation and the hosting state adopts the map in the same step. The argument selects one of three modes, each with a single insertion verb. With no argument or an iterable of entries, it is a plain keyed map (`map.Keyed<K, V>`, extending native `Map`). With a `State` class or parameterless factory, it is a pool (`map.Set<V, A>`, a native `Set` subclass): `add(...args)` spawns through the factory - class pools forward arguments to the constructor exactly as `Type.new()` accepts them - and returns the value, which is its own identity for `has`/`delete`/iteration. With a factory requiring its first parameter, it is a keyed spawning map (`map.Create<A, V>`, keyed by `A[0]`): `set(...args)` invokes the factory verbatim and stores the result at the first argument, replacing (and destroying, if owned) any previous value; there is no `add`. Spawning maps own what the factory makes - spawned `State` values are destroyed when deleted, cleared, or replaced - while values the factory passes through from its arguments stay guests. A map held by a `State` field adopts fresh members: parented to the owner, activated inside its context, and destroyed with it; the field is read-only, and a dead `State` value evicts itself from the map. `get()` with no key returns a shallow snapshot (`ReadonlyMap`, or `ReadonlySet` for pools); `keys(fn)` / `values(fn)` / `entries(fn)` return reusable transformed iterables.
